7 Preprocessing Transforms

Transform 1

Delta Encoding

Stores differences between consecutive bytes. Ideal for slowly-changing signals.

Transform 2

Delta-16

16-bit delta for structured data with 2-byte aligned values.

Transform 3

BWT + MTF

Burrows-Wheeler Transform + Move-to-Front for text-heavy data.

Transform 4

Run-Length Encoding

Collapses zero-runs. Perfect for sparse binary data.

Transform 5

Transpose

Column-wise reordering of structured records for better locality.

Transform 6

Nibble Split

Separates high and low nibbles. Reduces entropy in certain data patterns.

Transform 7

Identity

No preprocessing. Sometimes raw data compresses best on its own.

// Architecture

Block-by-Block Processing

Input Data
Raw bytes of any type
Accepts text, binary, structured, or mixed data
Adaptive Block Splitter
Entropy-guided variable block sizing
4KB–128KB blocks based on entropy variance
Block Profiler
Entropy, delta-entropy, null-ratio, structure detection
Computes 6 features: ent, de, zr, txt, struct, uniq
Candidate Selector
Context-aware preprocessing + backend pairing
Rules engine selects 3–15 candidates per block
Racing Engine
Try all candidates, smallest output wins
Per-block CRC32 integrity · HPRX binary format
HPRX Output
Self-contained compressed stream
16B global header + 14B per block header + data

// Quick Start

Three Lines of Code

from hyperpress import compress, decompress
 
compressed = compress(data)
restored = decompress(compressed)

Install with pip install hyperpress — Zero external dependencies. Pure Python. Python 3.8+.
